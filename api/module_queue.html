Module: queue

Mechanism for executing a series of callbacks in sequential order. Supports simple synchronous queueing as well as queuing callbacks across setTimeout. Callback iteration, specifying context and callback arguments, retrieval, removal, and promotion of queued callbacks. Pass in a configuration object with the same keys as a callback object (see the add method) to set the default values for those keys on all callbacks added to this Queue. E.g. Y.Queue({ timeout : 50 });

queue-io
Adds support for callback configuration key pauseForIOResponse. This will trigger the Queue to pause until all Y.io requests issued from inside the callback have received a response.
queue-base
Mechanism to execute a series of callbacks in sequence. Callbacks can be function references or object literals with the following keys:
fn
{Function} REQUIRED the callback function.
context
{Object} the desired execution context of the callback function
args
{Array} list of arguments to pass to the callback function
