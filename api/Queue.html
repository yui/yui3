Class Queue

Mechanism to execute a series of callbacks in sequence. Callbacks can be function references or object literals with the following keys:
fn
{Function} REQUIRED the callback function.
context
{Object} the desired execution context of the callback function
args
{Array} list of arguments to pass to the callback function

Constructor

Queue ( config , callback* )
Parameters:
config <Object> default callback configuration values
callback* <Function|Object> Any number of callbacks to initialize the queue

Properties

_ioAbortSub - protected {Object}

Event listener for global io:abort event to mark off io transactions, eventually restarting the Queue after all are complete or aborted.

_ioFailureSub - protected {Object}

Event listener for global io:success event to mark off io transactions, eventually restarting the Queue after all are complete or aborted.

_ioStartSub - protected {Object}

Event listener for global io:start event to cache active io transactions.

_ioSuccessSub - protected {Object}

Event listener for global io:success event to mark off io transactions, eventually restarting the Queue after all are complete or aborted.

_shiftSub - protected {Object}

Event listener on the Queue to trigger detaching event subscriptions for a given callback when that callback is shifted off the queue.

_tId - protected {Number}

Flag used to indicate the Queue is currently executing a callback.

_waiting - protected {Array}

Collection of outstanding io transaction ids that must complete before the Queue continues. Used during execution of callbacks configured with waitForIOResponse set to true.

active - {Boolean}

Indicates the execution state of the Queue.

defaults - protected {Object}

Callback defaults for this instance. Initially populated from the static Queue.defaults collection.

Queue.defaults - static {Object}

Defaults used to fill unset callback configuration values.

add

Queue add ( callback* )
Add any number of callbacks to the end of the queue. Callbacks passed in as functions will be wrapped in a callback object. Callbacks can be function references or object literals with these keys:
fn
{Function} REQUIRED the callback function.
name
{String} a reference name to use for promotion or access
context
{Object} the context from which to call the callback function.
timeout
{number} millisecond delay to wait after previous callback completion before executing this callback. Negative values cause immediate blocking execution. Default 0.
until
{Function} boolean function executed before each iteration. Return true to indicate callback completion.
iterations
{Number} number of times to execute the callback before proceeding to the next callback in the queue. Incompatible with until.
args
{Array} array of arguments passed to callback function
Parameters:
callback* <Function|Object> Any number of callbacks
Returns: Queue
the Queue instance
